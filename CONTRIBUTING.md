# المساهمة في مشروع Volt Pro

شكراً لاهتمامك بالمساهمة في مشروع Volt Pro! يرجى اتباع الإرشادات التالية.

## 📋 متطلبات قبل البدء

- معرفة بـ HTML, CSS, JavaScript
- معرفة بنظام Git
- فهم أساسي لنظام إدارة المشاريع

## 🔄 خطوات المساهمة

### 1. عمل نسخة من المشروع (Fork)
```bash
git clone https://github.com/voltprogroup/volt-pro-website.git
cd volt-pro-website
```

### 2. إنشاء فرع جديد
```bash
git checkout -b feature/your-feature-name
```

### 3. إجراء التغييرات
- قم بعمل تغييرات واضحة ومعقولة
- اختبر التغييرات جيداً قبل الإرسال
- تأكد من عدم كسر الميزات الموجودة

### 4. الالتزام بالتغييرات
```bash
git add .
git commit -m "وصف واضح للتغييرات"
```

### 5. دفع التغييرات
```bash
git push origin feature/your-feature-name
```

### 6. إنشاء طلب دمج (Pull Request)
- اشرح التغييرات بوضوح
- أرفق لقطات شاشة إن أمكن
- ارجع للقضايا ذات الصلة

## 🧪 الاختبار

### قبل إرسال الطلب:
- [ ] اختبر في متصفحات مختلفة
- [ ] تحقق من الاستجابة على الأجهزة المحمولة
- [ ] تأكد من عمل الميزات الجديدة
- [ ] تحقق من عدم وجود أخطاء في Console

## 🐛 الإبلاغ عن الأخطاء

عند الإبلاغ عن خطأ، تضمن:
- وصف واضح للمشكلة
- خطوات إعادة إنتاج الخطأ
- لقطة شاشة أو فيديو إن أمكن
- معلومات عن المتصفح والجهاز

```
العنوان: [النوع] وصف الخطأ
الوصف: شرح مفصل للمشكلة
الخطوات: 
1. ...
2. ...
المتوقع: ماذا يجب أن يحدث
الفعلي: ماذا يحدث الآن
```

## 🎨 طلبات الميزات الجديدة

قبل البدء بميزة جديدة، يرجى:
1. فتح issue لنقاشها
2. شرح الفائدة والاستخدام
3. انتظار الموافقة
4. تطوير الميزة
5. إرسال PR مع التوثيق الكامل

## 📚 قائمة التحقق النهائية

قبل إرسال PR:
- [ ] الكود يتبع معايير المشروع
- [ ] لا توجد أخطاء في Console
- [ ] الميزة مختبرة جيداً
- [ ] المستندات محدثة
- [ ] الالتزامات واضحة المعنى
- [ ] لا توجد ملفات حساسة (.env, credentials)
